Language Guide

Memory Management

Zith provides memory safety through explicit ownership semantics at the type level. There is no garbage collector — all memory management is compile-time enforced.

Ownership Keywords

Zith uses five ownership modifiers to control how values are accessed and shared:

Keyword Meaning
unique Sole owner of the value. No other references allowed.
shared Reference-counted shared ownership. Multiple owners allowed.
view Non-owning read-only reference (immutable borrow).
lend Non-owning mutable reference (mutable borrow).
extension Extends the lifetime of an owned value (used in APIs).

Unique Ownership

A unique value has exactly one owner. When the owner goes out of scope, the value is dropped:

zith
fn main() {
  let s: unique str = "hello";  // s owns the string
}  // s goes out of scope, memory freed

Move Semantics

Assigning a unique value transfers ownership:

zith
let s1: unique str = "hello";
let s2 = s1;  // Ownership moved to s2
// println(s1);  // Error! s1 no longer owns the value

View (Read-Only Reference)

Use view to borrow a value without taking ownership:

zith
fn len(s: view str): i32 {
  // Borrows s for reading, doesn't own it
}

let s: unique str = "hello";
len(s);  // Pass as view (implicit conversion)
// s still owns the string

Lend (Mutable Reference)

Use lend to borrow a value mutably:

zith
fn append(s: lend str, suffix: str) {
  // Can modify s
}

let mut s: unique str = "hello";
append(lend s, " world");  // Pass as lend

Shared Ownership

Use shared for reference-counted values that can have multiple owners:

zith
let a: shared str = "hello";
let b = a;  // Both a and b share ownership
// Value is dropped when last reference is gone

Borrowing Rules

  • One lend OR multiple view references at a time (not both)
  • References must not outlive the owner
  • All checks happen at compile time

Stack vs Heap

Stack Heap
Fixed size Dynamic size
Fast allocation Slower allocation
Automatic cleanup Managed via ownership
Primitives, arrays Strings, vectors, shared
ℹ

Zith's ownership model prevents data races and use-after-free errors at compile time without a garbage collector.
